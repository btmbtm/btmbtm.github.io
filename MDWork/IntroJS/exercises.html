    1. From your understanding of expressions and statements, write the difference between them.

      A statement is a discrete block of code that can be acted on by the interpreter (or ignored int he case of a comment), be it a simple declaration of a variable or a complex expression coupled with a function call.

      An expression is any combination of literals, variables, and operators that results in a value. Its results can be used by operations in a statement, such as the declaration of a variable or the calling of a function.

    1. Experiment with the precedence of arithmetic and logical operators. Write a short report of your findings.

      In Javascript, arithmetic operators follow PEMDAS from left to right, so any expressions within parenthesis are executed first, followed by function calls, followed by exponentiation, multiplication, division, and modular division, followed finally by division and subtraction.

      The logical NOT(!) evaluates before any of the expressions above. After any arithmetic operations are evaluated, less than type statements (<, <=)or more than type statements (>, >=) are evaluated first, followed by equality statements (==, ===) and inequality statements (!=, !===). AND statements (&&) are executed next, followed by any OR (||) statements.

      function gcd(A,B){
  if(A === 0) return B;
  if(B === 0) return A;
  if(B === A) return A;

  var big = Math.max(A,B);
  var small = Math.min(A,B);

  return gcd(small, big%small);

  var temp;
  for(var i = arr.length-1; i >= 0; i--){
    var nextLoc = Math.floor(Math.random()*i);
